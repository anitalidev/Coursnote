Transport Layer

TCP and UDP are the two transport protocols. They make opposite tradeoffs — understanding when to use each requires knowing why they differ.

The Core Question: Process to Process

IP addresses get a packet to the right computer. But a computer runs many processes at once (browser, Spotify, email). The transport layer uses port numbers to identify which process should receive each packet.

Socket = IP Address + Port Number

A socket is the unique identifier for a network endpoint — like an apartment address (IP) + unit number (port).

TCP — Transmission Control Protocol

"Like walking across the room before talking to someone — takes longer, but you know they heard you."

  • Connection-oriented: Must establish a connection (3-way handshake) before sending data
  • Reliable delivery: Guarantees all packets arrive, in order
  • Flow control: Doesn't overwhelm the receiver
  • Congestion control: Slows down if network is congested
  • Cost: Higher overhead, higher latency

Use when: Data loss is unacceptable — web pages, file downloads, email, SSH

UDP — User Datagram Protocol

"Like shouting at someone across the room — no guarantee they heard, but if they did, it was instant."

  • Connectionless: Just fire packets and hope for the best
  • No reliability: Lost packets stay lost — no retransmission
  • No ordering: Packets may arrive out of order
  • No congestion control: Sends at full rate regardless
  • Benefit: Very low overhead, very low latency

Use when: Speed matters more than perfection — video calls, gaming, live streams, DNS

Why Does UDP Exist If TCP is More Reliable?

TCP's reliability mechanisms add latency. In a video call, a 200ms-late packet is worse than no packet — by the time it arrives, the speaker has moved on. It's better to just skip it. DNS queries are tiny and need responses in milliseconds — the overhead of TCP's handshake would double the time. For these use cases, you want to control reliability at the application layer, not blindly use TCP.

How to Choose?

Use TCP when…

  • Data loss is unacceptable
  • Order matters (file content, web pages)
  • You don't need real-time delivery

Use UDP when…

  • Latency matters more than completeness
  • You can tolerate or handle loss yourself
  • You need low overhead (DNS, small queries)
