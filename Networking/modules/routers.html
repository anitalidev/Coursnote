Routers vs Switches

Both routers and switches move data — but they operate at different layers of the network stack and solve different problems.

Why two devices?

Networks have two distinct addressing problems. Inside a local network you need to find a specific machine by its hardware identity. Between different networks you need to know which network to send traffic toward. These require different address spaces, different lookup tables, and different logic — so networking split them into two specialised devices.

Switches — the LAN organiser

A switch connects devices within the same local area network (LAN). It operates at the Link Layer (Layer 2) and uses MAC addresses to decide where to forward a frame.

  • Every network interface has a burned-in 48-bit MAC address (e.g. a4:c3:f0:12:34:56).
  • A switch maintains a MAC address table: it learns which MAC address is reachable out of which port by watching incoming frames.
  • When a frame arrives, the switch looks up the destination MAC and forwards it only out the correct port — no broadcast storm.
  • Switches are fast because the lookup is simple and hardware-accelerated.
  • A switch has no concept of IP addresses — it cannot send traffic outside its LAN.

How a switch learns

  1. Frame arrives on Port 2 from MAC AA:BB:…
  2. Switch records: AA:BB:… → Port 2
  3. Switch looks up destination MAC in table
  4. If found → forward out that port only
  5. If not found → flood all ports except source

Routers — the inter-network navigator

A router connects different networks together. It operates at the Network Layer (Layer 3) and uses IP addresses to decide where to forward a packet.

  • IP addresses are logical and hierarchical (e.g. 192.168.1.0/24 is a subnet). This hierarchy enables efficient routing across the globe.
  • A router maintains a forwarding table (routing table): each entry maps a destination IP prefix to a next-hop router or outgoing interface.
  • When a packet arrives, the router performs longest-prefix matching to find the best route.
  • Routers decrement the IP TTL on every hop and discard packets whose TTL reaches 0 — preventing infinite loops.
  • A router does understand Link Layer (it must unwrap and rewrap frames at each hop), but its forwarding decision is always based on IP.

Longest-prefix match

Packet destined for 172.16.5.3:

  • 0.0.0.0/0 — matches (0 bits)
  • 172.16.0.0/16 — matches (16 bits) ✓ winner

The most specific (longest) matching prefix wins. This is how the internet routes billions of destinations with finite table entries.

Why not one device?

MAC addresses are flat — there is no structure telling you which port a far-away address is behind. Scaling MAC lookup to billions of devices is impossible. IP addresses are hierarchical: prefixes aggregate millions of hosts behind a single route. Routers exploit this hierarchy; switches cannot. Keeping the two layers separate lets each do what it is good at.
