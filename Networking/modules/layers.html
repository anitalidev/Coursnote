Protocol Layers

Protocols are organized into layers, where each layer handles a specific job and relies on the layer below. Click any layer to learn more.

Why Layer at All?

Layering is like building with Lego bricks instead of sculpting one piece. Advantage: you can swap out one layer's implementation without touching the others. Replace WiFi with Ethernet — the IP layer above doesn't care. Disadvantage: sometimes a lower layer has information a higher layer needs, and layering forces you to work around that; there's also duplication of some functions across layers.

Click each layer to expand it:

App
5 · Application Layer
HTTP, DNS, SMTP, FTP
Job: Where network applications live — your browser, email client, etc.

What it answers: "What do you want to do?" — GET a webpage, send an email, look up a name.

Unit of data: called a message

Examples: HTTP request (GET /index.html HTTP/1.1), DNS query for google.com, SMTP email.

Key insight: The application layer doesn't care how the data gets there — it just hands off to the transport layer below.
Transport
4 · Transport Layer
TCP, UDP — Port Numbers
Job: Get data from one process to another process on a different machine.

What it answers: "Which app on the destination should receive this?" — identified by a port number.

Unit of data: called a segment

Examples: Port 80 = HTTP, Port 443 = HTTPS, Port 25 = SMTP.

Key insight: IP addresses find the computer; port numbers find the application on that computer.
Network
3 · Network Layer
IP — IP Addresses
Job: Move packets from the source host all the way to the destination host across multiple networks.

What it answers: "Which computer (globally) is the destination?" — identified by an IP address.

Unit of data: called a datagram

Key insight: Routers only need to look at the network layer and below. The IP address stays constant end-to-end — it's the final destination. The MAC address changes at each hop.
Link
2 · Link Layer
Ethernet, WiFi — MAC Addresses
Job: Transfer a packet between two directly connected devices on the same link.

What it answers: "Which physical device on this local network do I deliver to next?" — identified by a MAC address.

Unit of data: called a frame

Key insight: MAC addresses are hardware-assigned and fixed. They're used to get a packet to the next hop only — each router replaces the destination MAC with the next router's MAC.
Physical
1 · Physical Layer
Cables, Fiber, Radio Waves
Job: Transmit raw bits over the physical medium.

How: Electrical signals on copper wire, light pulses on fiber optic, radio waves for WiFi.

Key insight: This is the only layer that deals with actual physics. All the "intelligence" lives above it. Different physical media can carry the same data — the layers above are unaffected.

Why Routers Don't Need Layers 4–5

A router's job is purely forwarding packets to the next hop. It doesn't care if you're watching YouTube or sending email — that's the application's business. By only reading up to Layer 3, routers can process packets much faster. This separation is key to internet scalability.
